Description

For every program a unique encryption key is generated. All registration keys are encrypted on the basis of the encryption key specified in this property. If you change it, then all previous keys of the object Trial become invalid for the new ID user generated with the help of a new encryption key. ID program shall also remain the same.

Remark

If you use two objects Trial - for example, one for activation key input and the other for generation of these keys (like in the example Examples\Difficult examples\Trial) then make sure that their ID program and encryption key are the same!
