Description

The useful object Object events makes accessible the properties and commands of that object where the event is being executed at the moment.

Example:

There Object events makes accessible the properties and commands of the Button1, because it is in the event Click of this button.

Thus the useful object Object events has nothing new, it just simplifies the process of actions creation. Now in case it is necessary to change the property of the object where the event has been executed, there is no need to specify its name (e.g. Button1), but simply a universal name "Object events" can be specified. This is very helpful, for example, when arrays are used.

All properties which are possible under different events and different objects are included in Object events in advance. That's why not all properties and commands are available under every particular event, but only properties and commands of the object where the given event has been executed. In the example only those properties and commands of Object events work, which are included in the object Button1 (Text in the given example).

Features of the object:
Properties and Commands Properties are settings of objects (color, text, size). By changing them you can set the object in a way you need. Objects can also execute some actions which are called commands and relate to them. You can change properties manually in Properties panel or program their changes in Actions tab.
