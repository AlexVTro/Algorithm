Description

Одним из замечательных инструментов, упрощающих разработку сложных программ является массив. Массив - это несколько объектов, объединенных под одним Именем, но имеющих разный Номер, что позволяет события всех объектов обрабатывать вместе.

Remark

Массивы мы будем рассматривать на примере Калькулятора. Этот проект находится в папке где установлен Алгоритм 2, в директории Проекты/Примеры/Калькулятор. Можете запустить его и опробовать все возможности массивов на нем, в процессе чтения данного материала.

Когда в вашей программе много различных объектов (кнопки, надписи) выполняют примерно одинаковые действия, их будет очень удобно объединить в массив, что позволит не создавать для каждого объекта отдельно события (клик для Кнопки1, клик для Кнопки2), а создавать общие события для всех объектов массива (Клик для Кнопки1 и Кнопки2). Примером массива могут служить кнопки с цифрами в проекте Калькулятор:

Событие Клик одно для всех кнопок с цифрами, т.к. у всех у них одно имя - "Кнопки". Это дало много преимуществ.

Во-первых это условие ЕСЛИ Окно1.ПамятьСтирания.Значение = "Надо очистить экран" ТОГДА. Оно должно выполняться для всех кнопок, и в данном случае нам не пришлось дублировать его в каждое событие, каждой кнопки с цифрой, если бы они не были объединены в массив.

Во-вторых мы очень элегантно вышли из положения, когда на экран дисплея надо вывести число, написанное на кнопке (это в последнем действии события - Окно1.Надпись1.Текст = Окно1.Надпись1.Текст & Окно1._Объект события.Текст). Ведь непонятно, на какую именно кнопку из массива нажали, если событие клик для всех них одно! Для таких случаев используется полезный объект с именем _Объект события. Он содержит свойства и комманды именно того объекта, над которым сейчас пользователь произвел событие.

Если же вы хотите добраться к какому-нибудь определенному объекту массива, то это можно сделать по свойству Номер:

Номера можно указвать цифрами - "1", "2", "3". Вспомогательными словом - Все, которое означает что свойство будет изменено у всех объектов массива. Так же можно использовать любые конструкции мастера сложных действий, т.е. значения любых свойств (например Окно1.Текст1.Ширина)

Чтобы пройтись по всем объектам используется цикл, а для определения, является ли объект четным, его номер делится на 2 (% - остаток от деления), и если номер поделился без остатка (остаток от деления = 0), значит объект четный.
