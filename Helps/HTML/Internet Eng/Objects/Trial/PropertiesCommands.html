Description

Properties are settings of objects (color, text, size). By changing them you can set the object in a way you need. Objects can also execute some actions which are called commands and relate to them.

You can change properties manually in Properties panel or program their changes in Actions tab. In the second case in the Events tab the event is created for the object and under this event the given property is changed. Examples of how to use properties and commands in Actions tab:

Window1.Object1.Text = "New text1"
Window1.Object2.Image = "C:\Pic.jpg"
Window1.Object2.Focus

List of object properties Trial:
_Cancel pressThis property is used to cancel a character that the user has just input. To cancel it change the property value to "Yes".
_Key char It's a literal symbol (a, b, c) which is input from the keyboard during the current event.
_Key pressed Contains the name of keyboard key (а, b, ctrl, enter) which is pressed during the event.
_Keyboard alt Displays whether the alt-key (Alt) is pressed at the moment or not.
_Keyboard control Displays whether the control-key (Ctrl) is pressed at the moment or not.
_Keyboard shift Displays whether the shift-key (Shift) is pressed at the moment or not.
_Mouse buttonStores the mouse button name (left button, scroll) with the help of which the event was called.
_Mouse clicks Keeps the number of clicks done to call the event.
_Mouse delta Keeps data about how many times the scroll was turned during the given event.
_Mouse X Keeps the current cursor's X coordinate . The property is available, for example, for the event "Mouse down", if that object in the objects list is selected for which the given event was created.
_Mouse Y Keeps the current cursor's Y coordinate . The property is available, for example, for the event "Mouse down", if that object in the objects list is selected for which the given event was created.
ActivationThe property indicates whether the program is activated or not, i.e. whether the activation key was input or not.
Activation cancelThis command cancels activation if the program has been activated earlier and the trial period is going on as though no activation was done.
AnchorAnchor helps to arrange objects in the window when sizes of window are changed by the user.
Back colorIt is the main color of the object.
Background imageIt is a background image of the first part of Double page.
Background image layoutThis is a style of the object background (Tile, Stretch).
Border styleSets the style of the object border.
Bring to frontThe command brings the object to front. This object locates uppermost of all other objects.
Context menuA menu which name was specified in this property is displayed after clicking of the object with the right button.
CursorIt is a symbol the mouse pointer is changed to when it is moved over the object.
Days allIndicates the number of days a trial period can be set for.
Days leftIndicates how many days of the trial period are left.
DockStretches an object along a window border (left, bottom). This resizing remains (or the object is stretched even more) when the user changes window sizes.
Enabled At any time you can disable some objects. The disabled object is displayed in the window, but the user can do nothing with it (e.g. he cannot press it, edit text, change size, etc.).
FocusFocus makes one particular object active, so the user works exactly with it (e.g. the user can focus a text box and values of all buttons he presses are input into this text box).
HeightIt is the size of vertical part of the object in pixels.
Id program For every program a special unique ID program is generated. All registration keys are associated with the specific ID program. If you'd change ID, all keys associated with the old ID would become invalid for the new ID. The encryption key should also remain the same.
Id registry programID registry program defines where in the registry data about trial period of your program should be stored. All registration keys are valid for the paticular ID program. If you'd change it, the keys would become invalid for the new ID program. And encryption key should also remain the same.
Id userThis ID is generated for any computer on the basis of registry data. Software and hardware replacement in most cases has NO influence on ID, but exceptions can happen sometimes.
IndexAlong with name index is an identifying property of the object. This property helps to differentiate objects with same names (in arrays).
Key encryption For every program a unique encryption key is generated. All registration keys are encrypted on the basis of the encryption key specified in this property. If you change it, then all previous keys of the object Trial become invalid for the new ID user generated with the help of a new encryption key. ID program shall also remain the same.
Key issueThe property indicates whether the program is registered or not, i.e. whether a proper activation key has been unput or not.
Key validation The command checks whether the received registration key is proper.
Maximum heightDefines maximum available height of the object. If it's not necessary to limit the object height, then set "0" as the property value.
Maximum widthDefines maximum available width of the object. If it's not necessary to limit the object width, then set "0" as the property value.
Message invalidWhen the user is trying to activate a program and the registration key is invalid, the massage specified in this property is displayed to the user.
Message validWhen the user has successfully activated the program, the corresponding message is displayed to the user. This property stores the text of the message.
Minimum heightDefines minimum available height of the object. If it's not necessary to limit the object height, then set "0" as the property value.
Minimum widthDefines minimum available width of the object. If it's not necessary to limit the object width, then set "0" as the property value.
NameAll objects in Algorithm2 are addressed by names. Using this property you can unambiguously specify an object which you want to work with.
RefreshThe command redraws an object in the window.
RemarkRemarks are used for additional information about a trial period: attempts to crack the program or to change the date, ect.
Send to backThe command replaces an object to the background. The object is below all other objects.
Tab indexWhen Tab key is pressed all objects of the window are activated in turn (they become focused). The less the Tab number is, the later the object would be activated.
Tab stopWhen the Tab key is pressed all objects of the window are activated in turn (they become focused). This property controls whether an object should be activated in its turn or skipped.
TagThis property doesn't affect the object. You can specify there anything you want, it is destined specially for you.
Text buttonA text specified in this property is displayed on the button.
Text fieldA text specified in this property is displayed on default in the text field for activation key input.
ToolTipWhen the cursor is moved over the object a tooltip specified in this field is displayed.
Trial start It starts counting down of trial period days.
Trial started It checks whether counting down of a trial period was started or not.
Type This property cannot be changed, it is destined to inform you about the type of the object (text box, button, window).
VisibleYou can make objects visible or invisible by setting this property.
WidthIt is the horizontal size of the object specified in pixels.
XCoordinate X of the object's upper left edge.
YCoordinate Y of the object upper left edge.
