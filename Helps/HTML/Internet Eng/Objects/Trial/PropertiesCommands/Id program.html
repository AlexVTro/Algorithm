Description

For every program a special unique ID program is generated. All registration keys are associated with the specific ID program. If you'd change ID, all keys associated with the old ID would become invalid for the new ID. The encryption key should also remain the same.

Remark

If you use two objects Trial, for example, one for activation key input and the other for generation of these keys (like in the example Examples\Difficult projects\Trial) then make sure that Id program and encryption key are the same!
