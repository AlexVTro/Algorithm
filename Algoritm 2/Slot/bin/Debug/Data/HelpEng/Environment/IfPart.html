Description

Terms allows you to specify whether to perform at the moment of action or not. The condition is a rule, usually comparison (for example, text box is "Window1" or width button above 100) and all action that come after this condition will be executed only when a condition is true O (ie, text box is really "Window1" a width button above 100)

Create condition You can go to the section conditions Development Environments Algorithm 2 :

It should introduce itself condition in the form : "Something" is equal/unequal "something" (unequal indicated <>, more or is marked >=)

otherCases:
They are checked in the event that the usual condition was wrong . If otherCase is correct, then it action executed. If not, you will be viewed following otherCases. Created otherCases the same way as usual conditions.

In all other cases:
This section provides for cases where the condition and (if available) otherCases prove incorrect. All his actions will be executed only if the condition does not work.

Remark

conditions may be not only simple (a comparison of the text, numbers) and complex (mathematical formula, the comparison with the value of the properties of another object). To create complex actions, use Master complex actions (the button with the wand on the right)
